# UFS 智驾系统测试计划 V1.0

**文档版本**: V1.6  
**创建时间**: 2026-03-13  
**修订时间**: 2026-03-13  
**测试对象**: 车载 UFS 存储系统  
**适用场景**: 智能驾驶系统存储验证（地平线 J6M 平台）  
**目标平台**: 地平线 J6M 智驾版，Linux 操作系统

---

## 📌 一、测试目标

本测试计划旨在全面验证 UFS 在智驾系统环境下的：
- **性能表现**：稳态性能、延迟特性、缓存效率
- **功能完整性**：UFS 标准功能、Multi-LUN、电源管理
- **可靠性**：上下电、异常掉电、长时间运行
- **环境适应性**：高低温、振动、车载工况

---

## 📌 二、测试核心思想

### 2.1 性能测试设计理念

> **关键洞察**：性能测试不只是测"快不快"，更要测"为什么快/慢"

- **稳态性能**：避免瞬时性能假象，验证长时间负载下的真实表现
- **空闲梯度分析**：通过空闲时间扫描（20ms~30s），反推固件后台任务触发机制
- **缓存分析**：分离 SLC Cache 和 Data Cache 的效果，通过性能对比间接推断缓存效果
- **QoS 延迟**：智驾对延迟敏感，关注尾延迟（P99/P99.99）而非平均延迟

### 2.2 可靠性测试设计理念

> **关键洞察**：车载环境的特殊性决定了可靠性测试必须覆盖极端工况

- **上下电循环**：模拟车辆频繁启动/熄火场景
- **异常掉电**：模拟车祸、电瓶故障等意外断电
- **电源状态切换**：验证低功耗模式下的功能完整性
- **长时间运行**：30 天连续运行，验证无性能衰减、无掉盘

### 2.3 固件算法分析理念

> **关键洞察**：通过性能表现反推固件算法策略

| 触发阈值 | 恢复时间 | 性能下降幅度 | 固件策略推断 |
|----------|----------|--------------|--------------|
| 20~40ms | <1s | >30% | **非常激进**：快速启动、快速完成，优先长期性能 |
| 40~60ms | 1~5s | 20~30% | **激进**：较快启动后台整理 |
| 60~100ms | 5~10s | 10~20% | **适中**：平衡响应速度和整理效率 |
| 100ms~1s | 10~30s | <10% | **保守**：延迟启动，优先响应速度 |
| >1s | >30s | 微小 | **非常保守**：Lazy GC 策略 |

## 📌 三、测试内容详解

### 3.1 性能测试模块

#### 3.1.1 基础性能测试

| 测试项 | 测试方法 | 关键指标 | 验收标准 |
|--------|----------|----------|----------|
| 顺序读取 | 128K~1MB 块，QD1/4/8 | 持续读取速度 (MB/s) | ≥标称值 95% |
| 顺序写入 | 128K~1MB 块，QD1/4/8 | 持续写入速度 (MB/s) | ≥标称值 90% |
| 随机读取 | 4K 块，QD1/4/8/16/32 | IOPS、延迟 (us) | IOPS≥标称值 90% |
| 随机写入 | 4K 块，QD1/4/8/16/32 | IOPS、延迟 (us) | IOPS≥标称值 90% |
| 混合负载 | 70% 读+30% 写，4K 随机 | IOPS、延迟分布 | 满足智驾场景需求 |
| **QoS 延迟分布** | 4K 随机，统计 P99/P99.99 | 尾延迟 (us) | P99≤2 倍平均延迟 |

#### 3.1.2 稳态性能测试

| 测试项 | 测试方法 | 观察重点 | 验收标准 |
|--------|----------|----------|----------|
| 长时间顺序写入 | 连续 4 小时满负载顺序写入 | 性能曲线、平均速度、波动范围 | 稳态≥标称 90%，波动≤15% |
| 长时间混合负载 | 连续 4 小时 7:3 随机读写 | IOPS 稳定性、延迟变化 | 无周期性掉速、无 IO 错误 |
| **满盘性能测试** | 磁盘使用 90% 后重复上述测试 | 性能衰减程度 | 满盘性能≥空盘 80% |

#### 3.1.3 空闲/活跃性能分析（固件算法反推）

**测试目的**：通过空闲时间梯度扫描，反推固件后台任务触发机制

**测试流程**：
```
步骤 1：建立活跃状态 → 连续写入 1 小时

步骤 2：空闲梯度扫描（关键区间加密）
  
  【关键区间】20ms~100ms（固件后台任务触发敏感区，10ms 步长）
  20ms → 30ms → 40ms → 50ms → 60ms → 80ms → 100ms
  
  【中等区间】100ms~1s（验证任务执行过程）
  200ms → 500ms → 1s
  
  【秒级区间】1s~30s（验证任务完成时间）
  2s → 5s → 10s → 30s
  
  【恢复验证】验证性能完全恢复
  1min → 10min

步骤 3：分析性能拐点
  - 性能开始下降点 → 后台任务触发阈值
  - 性能最低点 → 后台任务执行峰值
  - 性能恢复到 95% 点 → 后台任务完成时间
```

**总计测试点**：16 个

#### 3.1.4 Data Cache 性能测试

**注意**：
- 缓存命中率无法直接测量（固件黑盒）
- 缓存开关通常不支持（固件自动管理）

| 测试项 | 测试方法 | 验收标准 |
|--------|----------|----------|
| 缓存大小测试 | 写入不同大小文件 (128MB~4GB)，立即读取对比速度 | 缓存内读取速度≥写入速度 3 倍 |
| 冷热数据对比 | 首次随机读取 vs 重复读取同样数据 | 热数据延迟≤冷数据 50% |
| 缓存刷新机制 | 写入后不同时间点读取，观察性能衰减 | 明确刷新时间窗口 |
| 缓存开关对比 | **可选**：如固件支持配置 | 缓存开启提升≥30% |

**关键指标**：缓存容量边界、缓存内读取速度提升比例、缓存刷新时间窗口、热数据延迟降低比例

#### 3.1.5 SLC 缓存性能测试

| 测试项 | 测试方法 | 验收标准 |
|--------|----------|----------|
| SLC Cache 容量 | 连续写入直至速度骤降 | 符合规格书 |
| 缓内写入速度 | SLC 容量内连续写入 | ≥标称值 90% |
| 缓外写入速度 | 超出 SLC 容量后连续写入 | - |
| 性能衰减曲线 | 从空盘到写满的完整速度曲线 | 无明显断崖式下跌 |
