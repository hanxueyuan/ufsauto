## 🔧 适配后的开发工具和语言选型（适配精简环境）
### 最终选型：**纯 Bash 脚本开发，零依赖**
#### 原因：
1. 环境没有 Python，所有脚本必须用纯 Bash 编写，不依赖任何解释器
2. 尽量使用系统自带的命令，避免安装额外软件
3. 测试工具优先使用静态编译版本，避免依赖问题

### 允许使用的命令（经过验证）：
- ✅ 基础命令：`ls`、`cd`、`pwd`、`cp`、`mv`、`rm`、`mkdir`、`touch`
- ✅ 文件操作：`dd`、`cat`、`head`、`tail`、`md5sum`、`sha256sum`
- ✅ 系统信息：`uname`、`dmesg`、`df`、`free`、`top`
- ✅ 文本处理：`grep`、`awk`、`sed`、`sort`、`wc`
- ❌ 禁止使用：`python`、`perl` 等解释型语言
- ❌ 禁止使用：需要 apt 安装的工具（除非提供静态编译版本）

### 测试工具适配方案：
| 工具 | 适配方案 |
|------|----------|
| **fio（性能测试）** | 使用静态编译的 aarch64 版本 fio，直接上传到板上运行，无需安装 |
| **stress-ng（压力测试）** | 使用静态编译的 aarch64 版本，或用 `dd` + 循环脚本替代基础压力测试 |
| **ufs-utils（UFS专用工具）** | 静态编译 aarch64 版本，上传运行 |
| **性能监控** | 使用自带的 `iostat`、`vmstat`，或自行编写 Bash 监控脚本 |
| **报告生成** | 测试结果输出为文本/Markdown格式，在PC上生成可视化报告，不在板上运行 |

### 第一层：系统级基础验证
#### 1. 系统识别测试（纯Bash实现）
```bash
#!/bin/bash
echo "=== UFS 系统识别测试 ==="
echo "1. 内核版本：$(uname -r)"
echo "2. 架构：$(uname -m)"
echo "3. 块设备信息："
lsblk
echo "4. UFS 相关内核日志："
dmesg | grep -i ufs
echo "5. 磁盘使用情况："
df -h
```
**验证点**：
- UFS 设备正常识别，容量正确
- 内核日志无 UFS 相关错误
- 文件系统挂载正常

### 第二层：性能测试（使用静态编译fio）
```bash
# 顺序读测试
./fio --name=seq-read --filename=/home/ufs_test/test.img --ioengine=libaio --rw=read --bs=128k --numjobs=1 --iodepth=32 --runtime=60 --direct=1 --output=seq-read.log

# 顺序写测试
./fio --name=seq-write --filename=/home/ufs_test/test.img --ioengine=libaio --rw=write --bs=128k --numjobs=1 --iodepth=32 --runtime=60 --direct=1 --output=seq-write.log
```
**验证指标**：
- 顺序读 ≥2100MB/s
- 顺序写 ≥1600MB/s
- 4K随机读 ≥400K IOPS
- 4K随机写 ≥300K IOPS

## 🚀 部署方案（零依赖，直接运行）
### 1. 测试包结构
```
ufs_test_package/
├── bin/                # 静态编译的工具（fio、ufs-utils等）
├── tests/              # 测试脚本（纯Bash）
│   ├── 01_system_check.sh
│   ├── 02_function_test.sh
│   ├── 03_performance_test.sh
│   └── 04_reliability_test.sh
├── config/             # 测试配置
├── run_all.sh          # 一键执行脚本
└── README.md           # 使用说明
```

### 2. 执行方式
```bash
# 1. 上传测试包到开发板
scp ufs_test_package.tar.gz root@192.168.195.3:/home/

# 2. 解压
tar zxf ufs_test_package.tar.gz
cd ufs_test_package

# 3. 一键执行所有测试
./run_all.sh

# 4. 查看结果
cat results/report.md
```

### 3. 结果处理
- 测试结果在板上生成文本格式报告
- 将报告下载到PC上，生成可视化HTML/PDF报告
- 所有测试数据归档保存
