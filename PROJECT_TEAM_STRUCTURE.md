## 🐍 Python 依赖管理（最小化原则）
**严格控制Python依赖，仅使用标准库和最基础的第三方库**：
| 库名 | 用途 | 是否需要安装 | 备注 |
|------|------|--------------|------|
| os、sys、subprocess、shutil、re、json、datetime等 | 基础功能 | ✅ 标准库，无需安装 |
| markdown | 生成Markdown报告 | ⚠️ 可选，若无法安装则直接生成纯文本 |
| jinja2 | HTML报告模板 | ⚠️ 可选，若无法安装则跳过HTML报告 |
| matplotlib | 性能图表生成 | ⚠️ 可选，若无法安装则跳过图表 |

**降级方案**：
- 所有第三方库依赖都是可选的
- 若无法安装，自动降级为纯文本报告
- 核心功能完全不依赖任何第三方库，仅使用Python标准库即可运行
- 最大程度适配离线环境

## 🔄 GitHub 同步机制
1. **实时同步**：重要变更立即推送
2. **每日同步**：每日01:00自动同步所有变更
3. **版本管理**：每个里程碑打版本标签
4. **分支策略**：
   - `master`：稳定分支，经过测试验证
   - `develop`：开发分支，日常开发提交
   - `feature/*`：功能分支，新功能开发
   - `hotfix/*`：热修复分支，紧急问题修复
