## 1️⃣ 第 14 章：互操作性（Interoperability）

### 1.1 设备兼容性要求

**UFS 设备必须满足的兼容性要求**：

| 要求类别 | 具体要求 | 验证方法 |
|----------|----------|----------|
| **协议兼容性** | 支持 UFS 3.1 协议栈 | 协议一致性测试 |
| **电气兼容性** | 符合 M-PHY v4.0 电气规范 | 电气测试 |
| **命令兼容性** | 支持必需 SCSI/UFS 命令 | 命令测试 |
| **电源兼容性** | 支持 5 种电源模式 | 功耗测试 |
| **时序兼容性** | 符合时序要求 | 时序测试 |

### 1.2 主机兼容性要求

**UFS 主机必须满足的要求**：

| 要求类别 | 具体要求 | 说明 |
|----------|----------|------|
| **协议支持** | 支持 UFS 协议栈 | L1-L4 完整支持 |
| **命令支持** | 支持标准 SCSI 命令集 | READ/WRITE/QUERY 等 |
| **电源管理** | 支持设备电源状态切换 | ACTIVE/HIBERN8 等 |
| **错误处理** | 正确处理设备错误 | 超时/重试/恢复 |
| **多 LUN 支持** | 支持最多 8 个 LUN | 独立管理 |

## 2️⃣ 第 15 章：UFS 主机控制器接口（UFSHCI）

### 2.1 主机控制器架构

**UFSHCI 层次结构**：

```
┌─────────────────────────────────────────────────────────┐
│                    软件层（驱动）                        │
├─────────────────────────────────────────────────────────┤
│                 UFSHCI 主机控制器                        │
│  ┌─────────────┬─────────────┬─────────────────────┐   │
│  │ 命令队列     │ 数据路径     │ 中断处理             │   │
│  │ Command Queue│ Data Path   │ Interrupt Handler   │   │
│  └─────────────┴─────────────┴─────────────────────┘   │
├─────────────────────────────────────────────────────────┤
│                    M-PHY/UniPro                          │
├─────────────────────────────────────────────────────────┤
│                    UFS 设备                              │
└─────────────────────────────────────────────────────────┘
```

### 2.2 寄存器定义

**UFSHCI 主要寄存器**：

| 寄存器 | 偏移 | 功能 | 访问 |
|--------|------|------|------|
| UFSHC_CAP | 0x00 | 主机控制器能力 | RO |
| UFSHC_VER | 0x04 | 版本信息 | RO |
| UFSHC_HCE | 0x08 | 主机控制器使能 | RW |
| UFSHC_IS | 0x10 | 中断状态 | RW1C |
| UFSHC_IE | 0x14 | 中断使能 | RW |
| UFSHC_UTRLDBR | 0x18 | 传输请求列表门铃 | RW |
| UFSHC_UTRLCLR | 0x1C | 传输请求列表完成 | RO |
| UFSHC_UTMRLDBR | 0x20 | 任务管理门铃 | RW |
| UFSHC_UTMRLCLR | 0x24 | 任务管理完成 | RO |
| UFSHC_UTRLBA | 0x28 | 传输请求列表基地址 | RW |
| UFSHC_UTMRLBA | 0x30 | 任务管理列表基地址 | RW |

### 2.3 命令提交流程

**命令提交步骤**：

```
1. 驱动准备命令描述符（Command Descriptor）
   - 设置命令类型（READ/WRITE/QUERY）
   - 设置 LBA、长度等参数
   - 设置数据缓冲区地址

2. 写入传输请求列表（Transfer Request List）
   - 将命令描述符写入内存
   - 更新列表指针

3.  ringing Doorbell（敲钟）
   - 写入 UFSHC_UTRLDBR 寄存器
   - 通知主机控制器有新命令

4. 主机控制器执行命令
   - 从内存读取命令描述符
   - 通过 M-PHY/UniPro 发送到设备
   - 等待设备响应

5. 中断处理
   - 设备完成命令后触发中断
   - 主机控制器更新完成寄存器
   - 驱动读取结果
```

**代码示例**：
```c
// 命令提交流程示例
void ufs_submit_command(struct ufs_hba *hba, struct ufs_command *cmd)
{
    // 1. 准备命令描述符
    cmd_desc = hba->utrdl + slot;
    prepare_command_descriptor(cmd_desc, cmd);
    
    // 2. 写入传输请求列表
    writel(cmd_desc_addr, hba->mmio + UFSHC_UTRLBA);
    
    // 3. Ring Doorbell
    writel(1 << slot, hba->mmio + UFSHC_UTRLDBR);
    
    // 4. 等待完成（中断或轮询）
    wait_for_completion(&hba->cmd_complete);
    
    // 5. 读取结果
    result = read_completion_status(hba, slot);
}
```

### 2.4 中断处理

**中断类型**：

| 中断类型 | 位掩码 | 说明 |
|----------|--------|------|
| 传输请求完成 | 0x0001 | 命令执行完成 |
| 任务管理完成 | 0x0002 | 任务管理完成 |
| 设备错误 | 0x0010 | 设备报告错误 |
| 主机错误 | 0x0020 | 主机内部错误 |
| 电源管理事件 | 0x0100 | 电源状态变化 |

**中断处理流程**：
```
1. 设备触发中断
2. 主机控制器设置中断状态寄存器（UFSHC_IS）
3. CPU 响应中断，调用中断处理函数
4. 读取中断状态，判断中断类型
5. 根据类型处理：
   - 传输完成 → 唤醒等待的驱动
   - 设备错误 → 错误恢复
   - 电源事件 → 电源状态更新
6. 清除中断状态（写 1 清除）
```

## 3️⃣ 附录 A：错误码定义

### 3.1 UFS 错误码

| 错误码 | 值 | 说明 |
|--------|-----|------|
| UFS_SUCCESS | 0x00 | 成功 |
| UFS_ERROR_GENERAL | 0x01 | 一般错误 |
| UFS_ERROR_TIMEOUT | 0x02 | 超时错误 |
| UFS_ERROR_INVALID_CMD | 0x03 | 无效命令 |
| UFS_ERROR_INVALID_LBA | 0x04 | 无效 LBA |
| UFS_ERROR_NO_SPACE | 0x05 | 空间不足 |
| UFS_ERROR_DEVICE_FAILURE | 0x06 | 设备故障 |
| UFS_ERROR_POWER_FAILURE | 0x07 | 电源故障 |

### 3.2 错误处理策略

**错误恢复流程**：
```
1. 检测到错误
2. 记录错误日志
3. 根据错误类型处理：
   - 超时 → 重试（最多 3 次）
   - 无效命令 → 报告错误，不重试
   - 设备故障 → 尝试复位设备
   - 电源故障 → 重新上电初始化
4. 如果重试失败 → 报告上层
5. 更新健康状态（如适用）
```
