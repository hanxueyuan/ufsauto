## ⚡ 电源模式总览

UFS 3.1 支持 5 种电源模式，功耗逐级降低：

```
功耗等级（从高到低）：
┌─────────────────────────────────────────────────────────┐
│  模式       │  相对功耗  │  典型唤醒时间  │  应用场景    │
├─────────────────────────────────────────────────────────┤
│  ACTIVE     │   100%     │      -         │  正常工作    │
│  IDLE       │   ~30%     │    <10μs       │  空闲等待    │
│  SLEEP      │   ~10%     │    <50μs       │  睡眠模式    │
│  POWERDOWN  │   ~5%      │    <1ms        │  掉电模式    │
│  HIBERN8    │   ~1%      │    <10ms       │  休眠模式    │
└─────────────────────────────────────────────────────────┘
```

---

## 1️⃣ ACTIVE 模式

**特点**：
- 全功能工作状态
- 所有电路供电正常
- 可执行任意命令
- 功耗最高

**进入条件**：
- 上电后默认状态
- 从其他模式唤醒

**退出条件**：
- 空闲超时 → IDLE
- 主机请求 → SLEEP/HIBERN8

**测试要点**：
- [ ] ACTIVE 模式功耗测量
- [ ] 命令响应时间测试
- [ ] 满载功耗测试

---

## 2️⃣ IDLE 模式

**特点**：
- 设备空闲等待
- 部分电路降功耗
- 快速唤醒（<10μs）
- 可快速返回 ACTIVE

**进入条件**：
- 空闲超时（可配置）
- 主机请求

**退出条件**：
- 新命令到达 → 自动唤醒

**时序要求**：
```
ACTIVE → IDLE: 空闲超时后自动进入
IDLE → ACTIVE: <10μs（收到命令后）
```

**测试要点**：
- [ ] 空闲超时配置测试
- [ ] 唤醒延迟测量
- [ ] 功耗降低比例验证

---

## 3️⃣ SLEEP 模式

**特点**：
- 深度睡眠状态
- 大部分电路断电
- 唤醒时间<50μs
- 保持配置信息

**进入条件**：
- 主机发送进入 SLEEP 请求
- 设备同意进入

**退出条件**：
- 主机发送唤醒请求

**时序要求**：
```
ACTIVE → SLEEP: 
  1. Host 发送进入请求
  2. Device 确认
  3. 进入 SLEEP（<100μs）

SLEEP → ACTIVE:
  1. Host 发送唤醒请求
  2. Device 唤醒（<50μs）
  3. 恢复正常工作
```

**测试要点**：
- [ ] 进入/退出流程测试
- [ ] 唤醒延迟测量（<50μs）
- [ ] 配置信息保持验证
- [ ] 功耗测量（~10%）

---

## 4️⃣ POWERDOWN 模式

**特点**：
- 接近掉电状态
- 仅保留必要电路
- 唤醒时间<1ms
- 配置信息可能丢失

**进入条件**：
- 主机请求
- 长时间空闲

**退出条件**：
- 主机唤醒请求
- 硬件复位

**时序要求**：
```
ACTIVE → POWERDOWN: <1ms
POWERDOWN → ACTIVE: <1ms
```

**测试要点**：
- [ ] 进入/退出流程测试
- [ ] 唤醒延迟测量（<1ms）
- [ ] 配置信息保持验证
- [ ] 功耗测量（~5%）

---

## 5️⃣ HIBERN8 模式 ⭐ 重点

**特点**：
- 最低功耗模式
- M-PHY 断电
- UniPro 保持状态
- 唤醒时间<10ms
- 配置信息保持

**进入条件**：
- 主机发送 HIBERN8 请求（H8_Enter）
- 设备确认（H8_Exit）

**退出条件**：
- 主机发送 HIBERN8 退出请求（H8_Exit）
- 设备请求退出（H8_Exit_Indication）

**时序要求**：
```
ACTIVE → HIBERN8:
  1. Host 发送 H8_Enter
  2. Device 确认 H8_Enter_Confirm
  3. 进入 HIBERN8（<100μs）

HIBERN8 → ACTIVE:
  1. Host 发送 H8_Exit
  2. Device 唤醒 M-PHY
  3. 恢复正常工作（<10ms）
```

**状态转换图**：
```
                    ┌─────────────┐
                    │   ACTIVE    │
                    └──────┬──────┘
                           │
              ┌────────────┼────────────┐
              │            │            │
              ▼            ▼            ▼
        ┌──────────┐ ┌──────────┐ ┌──────────┐
        │   IDLE   │ │  SLEEP   │ │ HIBERN8  │
        └──────────┘ └──────────┘ └────┬─────┘
                                       │
                                       ▼
                                 ┌──────────┐
                                 │POWERDOWN │
                                 └──────────┘
```

**测试要点**：
- [ ] H8 进入/退出流程测试
- [ ] 唤醒延迟测量（<10ms）
- [ ] 配置信息保持验证
- [ ] 功耗测量（~1%）
- [ ] 循环测试（1000 次 H8 循环）

## 🔋 自适应电流控制（ICC）

**功能**：
- 根据负载动态调整电流
- 减少功耗和发热
- 支持多个 ICC 级别

**ICC 级别**：
| 级别 | 最大电流 | 应用场景 |
|------|----------|----------|
| ICC Level 0 | 最低 | 低功耗模式 |
| ICC Level 1 | 低 | 轻负载 |
| ICC Level 2 | 中 | 中等负载 |
| ICC Level 3 | 高 | 重负载 |
| ICC Level 4 | 最高 | 峰值负载 |

**控制方式**：
- 主机通过属性设置 ICC 级别
- 设备根据负载自动调整
- 可配置 ICC 超时时限

**测试要点**：
- [ ] ICC 级别切换测试
- [ ] 功耗测量（各 ICC 级别）
- [ ] 自动调整响应测试
- [ ] 超时机制测试

## 🔧 电源管理调试技巧

### 1. 功耗测量方法

```
测试设备：
- 高精度电流表（uA 级）
- 示波器（捕捉瞬态）
- 可编程电源

测量点：
- VCC 电流（主电源）
- VCCQ 电流（I/O 电源）
- VCCQ2 电流（辅助电源）
```

### 2. 唤醒时间测量

```
测试方法：
1. 示波器探头接电源引脚
2. 发送模式切换命令
3. 测量命令到恢复的时间

要求：
- IDLE 唤醒：<10μs
- SLEEP 唤醒：<50μs
- HIBERN8 唤醒：<10ms
```

### 3. 循环测试脚本

```bash
#!/bin/bash
# HIBERN8 循环测试脚本

for i in {1..1000}; do
    # 进入 HIBERN8
    echo "H8_Enter" > /sys/class/ufs/ufs0/power_state
    
    # 等待 100ms
    sleep 0.1
    
    # 退出 HIBERN8
    echo "ACTIVE" > /sys/class/ufs/ufs0/power_state
    
    # 验证功能
    fio --name=test --rw=read --bs=4k --size=1M --numjobs=1
    
    # 记录日志
    echo "Cycle $i completed" >> h8_test.log
done
```

## 📝 学习总结

### 核心要点
1. **5 种电源模式**：ACTIVE/IDLE/SLEEP/POWERDOWN/HIBERN8
2. **功耗逐级降低**：100% → 30% → 10% → 5% → 1%
3. **唤醒时间要求**：IDLE<10μs, SLEEP<50μs, HIBERN8<10ms
4. **HIBERN8 最重要**：车载场景常用，需重点测试
5. **ICC 自适应**：根据负载动态调整电流

### 测试应用
1. 基础模式测试验证各模式功能
2. 循环测试验证可靠性（1000 次）
3. 功耗测量验证节能效果
4. 唤醒延迟验证响应速度
