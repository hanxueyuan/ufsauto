# UFS 3.1 电源管理测试用例集

**文档版本**: V1.0  
**创建时间**: 2026-03-19  
**参考标准**: JEDEC JESD220E 第 9 章 - 电源管理  
**适用产品**: 群联 PS8363 + 长存 SQS 128GB UFS 3.1  
**用途**: 电源模式功能验证 + 车规级温度/可靠性验证

---

## 1. 概述

UFS 3.1 定义了 5 种电源模式（Active/Idle/Sleep/PowerDown/Hibernate），以及 ICC 自适应电流控制机制。本用例集针对车规级应用场景，覆盖基础模式测试、模式转换测试、唤醒时间测试、循环可靠性测试、温度极端条件测试和 ICC 控制测试。

**测试设备需求**：
- 高精度数字万用表（μA 级电流测量）
- 示波器（≥1GHz 带宽，捕捉唤醒瞬态）
- 可编程电源（VCC/VCCQ/VCCQ2 独立控制）
- 温箱（-40°C ~ 150°C，AEC-Q100 要求）
- UFS Host 测试板 + 逻辑分析仪

**通用前置条件**：
- UFS 设备已正常枚举
- 电流测量探头已校准
- 测试环境温度已稳定（±2°C）

## 2. 基础电源模式功能测试

### TC-PWR-001: ACTIVE 模式功耗基线

| 项目 | 内容 |
|------|------|
| **用例 ID** | TC-PWR-001 |
| **测试目标** | 建立 ACTIVE 模式功耗基线数据 |
| **前置条件** | 设备处于 ACTIVE 模式，室温 25°C |
| **测试步骤** | 1. 测量空闲状态 VCC 电流（无 I/O）<br>2. 执行顺序读取（128KB, QD=1），测量 VCC 电流<br>3. 执行顺序写入（128KB, QD=1），测量 VCC 电流<br>4. 执行随机混合读写（4KB, 70R/30W, QD=32），测量 VCC 电流<br>5. 同时测量 VCCQ 和 VCCQ2 电流<br>6. 计算总功耗 P = VCC×IVCC + VCCQ×IVCCQ + VCCQ2×IVCCQ2 |
| **预期结果** | 功耗在规格书标称范围内 |
| **Pass 标准** | 各场景功耗 ≤ 规格书最大值 |
| **Fail 标准** | 功耗超过规格书最大值 |
| **记录数据** | 空闲/顺序读/顺序写/随机混合 各场景的 VCC/VCCQ/VCCQ2 电流和总功耗 |

## 10. 测试脚本参考

```bash
#!/bin/bash
# HIBERN8 循环测试脚本
# 使用 sysfs 接口控制 UFS 电源状态

UFS_PATH="/sys/bus/platform/drivers/ufshcd"
DEVICE="/dev/sdX"
LOG_FILE="hibern8_cycle_test.log"
TOTAL_CYCLES=100000
CHECK_INTERVAL=1000

echo "Starting HIBERN8 cycle test: $TOTAL_CYCLES cycles" | tee $LOG_FILE
fail_count=0

for ((i=1; i<=TOTAL_CYCLES; i++)); do
    # 进入 HIBERN8
    echo "hibern8" > ${UFS_PATH}/*/power_mode 2>/dev/null
    usleep 100000  # 100ms
    
    # 退出 HIBERN8
    echo "active" > ${UFS_PATH}/*/power_mode 2>/dev/null
    
    # 定期功能检查
    if ((i % CHECK_INTERVAL == 0)); then
        # 数据完整性检查
        dd if=$DEVICE of=/dev/null bs=1M count=1 2>/dev/null
        if [ $? -ne 0 ]; then
            echo "FAIL at cycle $i: read error" | tee -a $LOG_FILE
            ((fail_count++))
        else
            echo "Cycle $i/$TOTAL_CYCLES: OK" | tee -a $LOG_FILE
        fi
    fi
done

echo "Test complete. Total: $TOTAL_CYCLES, Failed: $fail_count" | tee -a $LOG_FILE
```
