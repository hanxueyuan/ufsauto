|------|------|

## 二、性能测试 (5 条)

### TC-010: 顺序读写带宽
| 项目 | 内容 |
|------|------|
| **目的** | 测量顺序读写最大带宽 |
| **步骤** | `fio --name=seq --rw=read --bs=1M --size=4G` |
| **预期** | UFS 3.1: 读≥2000MB/s, 写≥1200MB/s |
| **车规关联** | 智驾数据记录带宽需求 |

### TC-011: 随机读写 IOPS
| 项目 | 内容 |
|------|------|
| **目的** | 测量 4K 随机读写 IOPS |
| **步骤** | `fio --name=rand --rw=randread --bs=4K --iodepth=64` |
| **预期** | UFS 3.1: 读≥60K IOPS, 写≥50K IOPS |
| **车规关联** | 系统启动/应用加载性能 |

### TC-012: 混合负载性能
| 项目 | 内容 |
|------|------|
| **目的** | 验证读写混合场景性能 |
| **步骤** | 70% 读 +30% 写混合负载测试 |
| **预期** | 性能下降≤20% |
| **车规关联** | 实际智驾工作负载 |

### TC-013: 延迟测试
| 项目 | 内容 |
|------|------|
| **目的** | 测量读写延迟分布 |
| **步骤** | `fio --lat_percentiles=1` 获取 P99 延迟 |
| **预期** | P99 读延迟≤50μs, 写≤100μs |
| **车规关联** | 实时系统延迟要求 |

### TC-014: Hibernation 唤醒延迟
| 项目 | 内容 |
|------|------|
| **目的** | 测量 H8 唤醒时间 |
| **步骤** | 1. 进入 H8 2. 触发唤醒 3. 记录恢复时间 |
| **预期** | 唤醒时间≤15μs |
| **车规关联** | 快速启动需求 |
