## 📋 5 种电源模式详解

### 模式总览

| 模式 | 功耗 | 唤醒时间 | M-PHY 状态 | UniPro 状态 | 应用场景 |
|------|------|----------|------------|-------------|----------|
| **ACTIVE** | 100% | - | ACTIVE | ACTIVE | 正常读写操作 |
| **IDLE** | ~30% | <10μs | STALL | ACTIVE | 短暂空闲等待 |
| **SLEEP** | ~10% | <50μs | HIBERN8 | SLEEP | 睡眠模式 |
| **POWERDOWN** | ~5% | <1ms | OFF | OFF | 掉电模式 |
| **HIBERN8** | ~1% | <10ms | HIBERN8 | HIBERN8 | 长时间休眠 |

## 1️⃣ ACTIVE 模式（活跃模式）

### 模式特点

**功能状态**:
- ✅ 所有电路正常供电
- ✅ M-PHY 处于 ACTIVE 状态
- ✅ UniPro 处于 ACTIVE 状态
- ✅ 可执行任意命令（READ/WRITE/QUERY 等）
- ✅ 支持全队列深度（QD1-QD32）

**功耗特性**:
| 操作类型 | VCC 电流 | VCCQ 电流 | 总功耗 |
|----------|----------|-----------|--------|
| 顺序读取 | ~200 mA | ~50 mA | ~700 mW |
| 顺序写入 | ~250 mA | ~50 mA | ~850 mW |
| 随机读取 | ~180 mA | ~50 mA | ~650 mW |
| 随机写入 | ~220 mA | ~50 mA | ~750 mW |

### 进入/退出条件

**进入 ACTIVE**:
- 上电后默认状态
- 从其他模式唤醒

**退出 ACTIVE**:
- 空闲超时 → IDLE
- 主机请求 → SLEEP/HIBERN8

| PWR-ACT-002 | ACTIVE 命令执行 | 执行 READ/WRITE/QUERY | 命令成功 |
| PWR-ACT-003 | ACTIVE 队列深度 | 测试 QD1-QD32 性能 | 性能符合预期 |

## 2️⃣ IDLE 模式（空闲模式）

### 模式特点

**功能状态**:
- ✅ 部分电路降功耗
- ✅ M-PHY 处于 STALL 状态
- ✅ UniPro 保持 ACTIVE
- ✅ 可快速唤醒（<10μs）
- ❌ 不能执行命令（需先唤醒）

**功耗特性**:
| 参数 | 值 | 说明 |
|------|-----|------|
| VCC 电流 | ~50 mA | 比 ACTIVE 降低 75% |
| VCCQ 电流 | ~20 mA | 比 ACTIVE 降低 60% |
| 总功耗 | ~200 mW | 比 ACTIVE 降低 70% |

### 进入/退出条件

**进入 IDLE**:
```
1. 主机检测到空闲超时（可配置，如 100ms）
2. 主机发送进入 IDLE 请求（可选）
3. 设备确认
4. 进入 IDLE 模式
```

**退出 IDLE**:
```
1. 新命令到达（自动唤醒）
2. 唤醒时间：<10μs
3. 恢复到 ACTIVE 模式
```

### 时序要求

| 参数 | 要求 | 单位 |
|------|------|------|
| 空闲超时 | 可配置（默认 100ms） | ms |
| 进入时间 | <1 | μs |
| 唤醒时间 | <10 | μs |

| 用例 ID | 测试项 | 测试方法 | 验收标准 |
|---------|--------|----------|----------|
| PWR-IDL-001 | IDLE 功耗测量 | 测量 IDLE 模式电流 | ~200 mW |
| PWR-IDL-002 | IDLE 唤醒延迟 | 发送命令测延迟 | <10μs |
| PWR-IDL-003 | 空闲超时配置 | 配置不同超时时间 | 配置生效 |
| PWR-IDL-004 | IDLE 循环测试 | 1000 次 ACTIVE↔IDLE 循环 | 功能正常 |

## 3️⃣ SLEEP 模式（睡眠模式）

### 模式特点

**功能状态**:
- ✅ 大部分电路断电
- ✅ M-PHY 处于 HIBERN8 状态
- ✅ UniPro 处于 SLEEP 状态
- ✅ 保持配置信息
- ❌ 不能执行命令

**功耗特性**:
| 参数 | 值 | 说明 |
|------|-----|------|
| VCC 电流 | ~5 mA | 比 ACTIVE 降低 97% |
| VCCQ 电流 | ~5 mA | 比 ACTIVE 降低 90% |
| 总功耗 | ~30 mW | 比 ACTIVE 降低 95% |

### 进入/退出流程

**进入 SLEEP**:
```
1. 主机发送进入 SLEEP 请求（Set Power Mode）
2. 设备确认（Response）
3. 设备进入 SLEEP 模式
4. 主机等待设备进入完成

时序：
- 进入时间：<10μs
```

**退出 SLEEP**:
```
1. 主机发送唤醒请求（Set Power Mode = ACTIVE）
2. 设备唤醒 M-PHY 和 UniPro
3. 设备确认唤醒完成
4. 恢复正常工作

时序：
- 唤醒时间：<50μs
- 时钟稳定：<10μs
```

### 时序图

```
主机                          设备
  │                            │
  │── Set Power Mode (SLEEP) ──│→ 设备确认
  │                            │→ 进入 SLEEP (<10μs)
  │                            │
  │                            │... 休眠中 ...
  │                            │
  │── Set Power Mode (ACTIVE) ─│→ 设备确认
  │                            │→ 唤醒 (<50μs)
  │                            │→ 时钟稳定 (<10μs)
  │←────── 准备就绪 ───────────│
```

## 4️⃣ POWERDOWN 模式（掉电模式）

### 模式特点

**功能状态**:
- ✅ 接近掉电状态
- ✅ M-PHY 关闭（OFF）
- ✅ UniPro 关闭（OFF）
- ✅ 仅保留必要电路
- ❌ 配置信息可能丢失

**功耗特性**:
| 参数 | 值 | 说明 |
|------|-----|------|
| VCC 电流 | ~1 mA | 比 ACTIVE 降低 99% |
| VCCQ 电流 | ~0 mA | 关闭 |
| 总功耗 | ~6 mW | 比 ACTIVE 降低 99% |

### 进入/退出流程

**进入 POWERDOWN**:
```
1. 主机请求进入 POWERDOWN
2. 设备保存必要状态
3. 关闭 M-PHY 和 UniPro
4. 进入 POWERDOWN 模式

时序：
- 进入时间：<1ms
```

**退出 POWERDOWN**:
```
1. 主机发送上电请求
2. 设备重新初始化 M-PHY/UniPro
3. 恢复配置（如需要）
4. 恢复正常工作

时序：
- 唤醒时间：<1ms
- 时钟稳定：<10ms
- 配置恢复：可能需要
```

### 与 SLEEP 的区别

| 特性 | SLEEP | POWERDOWN |
|------|-------|-----------|
| 功耗 | ~30 mW | ~6 mW |
| 唤醒时间 | <50μs | <1ms |
| 配置保持 | ✅ 保持 | ❌ 可能丢失 |
| 适用场景 | 短暂休眠 | 长时间掉电 |

| PWR-PDW-003 | POWERDOWN 功耗测量 | 测量 POWERDOWN 电流 | ~6 mW |

## 5️⃣ HIBERN8 模式（休眠模式）⭐ 重点

### 模式特点

**功能状态**:
- ✅ 最低功耗模式
- ✅ M-PHY 处于 HIBERN8 状态
- ✅ UniPro 处于 HIBERN8 状态
- ✅ 配置信息保持
- ❌ 不能执行命令

**功耗特性**:
| 参数 | 值 | 说明 |
|------|-----|------|
| VCC 电流 | ~1 mA | 比 ACTIVE 降低 99% |
| VCCQ 电流 | ~1 mA | 比 ACTIVE 降低 98% |
| 总功耗 | ~6 mW | 比 ACTIVE 降低 99% |

### 进入/退出流程（重点掌握）

**进入 HIBERN8**:
```
1. 主机发送 H8_Enter 请求（通过 DME 设置）
2. 设备确认 H8_Enter_Confirm
3. M-PHY 进入 HIBERN8 状态
4. UniPro 进入 HIBERN8 状态
5. 时钟关闭

时序：
- 进入时间：<100μs
```

**退出 HIBERN8**:
```
1. 主机发送 H8_Exit 请求（通过 DME 设置）
2. 设备唤醒 M-PHY
3. 设备唤醒 UniPro
4. 时钟稳定
5. 设备发送 H8_Exit_Confirm
6. 恢复正常工作

时序：
- 唤醒时间：<10ms
- 时钟稳定：<5ms
```

### HIBERN8 状态机

```
                    ┌─────────────┐
                    │   ACTIVE    │
                    └──────┬──────┘
                           │
              ┌────────────┼────────────┐
              │            │            │
              ▼            ▼            ▼
        ┌──────────┐ ┌──────────┐ ┌──────────┐
        │   IDLE   │ │  SLEEP   │ │ HIBERN8  │
        └──────────┘ └──────────┘ └────┬─────┘
                                       │
                                       ▼
                                 ┌──────────┐
                                 │POWERDOWN │
                                 └──────────┘

HIBERN8 转换:
- ACTIVE → HIBERN8: H8_Enter 请求
- HIBERN8 → ACTIVE: H8_Exit 请求
```

### HIBERN8 与 SLEEP 的区别

| 特性 | SLEEP | HIBERN8 |
|------|-------|---------|
| **功耗** | ~30 mW | ~6 mW |
| **唤醒时间** | <50μs | <10ms |
| **M-PHY 状态** | HIBERN8 | HIBERN8 |
| **UniPro 状态** | SLEEP | HIBERN8 |
| **配置保持** | ✅ 保持 | ✅ 保持 |
| **适用场景** | 短暂休眠（秒级） | 长时间休眠（分钟级以上） |
| **车载应用** | 等红灯/短暂停车 | 长时间停车 |

### 时序图（重点掌握）

```
主机                          设备
  │                            │
  │── DME_SET (H8_Enter) ──────│→ 设备确认
  │←── DME_GET (H8_Enter_Confirm) ─│
  │                            │→ M-PHY 进入 HIBERN8
  │                            │→ UniPro 进入 HIBERN8 (<100μs)
  │                            │
  │                            │... 休眠中 ...
  │                            │
  │── DME_SET (H8_Exit) ───────│→ 设备唤醒
  │                            │→ M-PHY 唤醒
  │                            │→ UniPro 唤醒
  │                            │→ 时钟稳定 (<5ms)
  │←── DME_GET (H8_Exit_Confirm) ─│
  │←────── 准备就绪 ───────────│ (<10ms)
```

## 📊 5 种模式对比总结

### 功耗对比

```
功耗（相对值）:
ACTIVE     ████████████████████ 100%  (~700 mW)
IDLE       ██████ ~30%           (~200 mW)
SLEEP      ██ ~10%               (~30 mW)
POWERDOWN  █ ~5%                 (~6 mW)
HIBERN8    █ ~1%                 (~6 mW)
```

### 唤醒时间对比

```
唤醒时间:
IDLE       █ <10μs      (最快)
SLEEP      ████ <50μs   (快)
POWERDOWN  ████████ <1ms (中等)
HIBERN8    ████████████████ <10ms (慢)
```

### 应用场景对比

| 场景 | 推荐模式 | 原因 |
|------|----------|------|
| 正常读写 | ACTIVE | 全功能支持 |
| 短暂空闲（<100ms） | ACTIVE | 避免频繁切换 |
| 空闲等待（100ms-10s） | IDLE | 快速唤醒 |
| 短暂停车（10s-1min） | SLEEP | 平衡功耗和唤醒 |
| 长时间停车（>1min） | HIBERN8 | 最低功耗 |
| 车辆维修/运输 | POWERDOWN | 接近掉电 |

## 📝 学习总结

### 核心要点
1. **5 种模式** - ACTIVE/IDLE/SLEEP/POWERDOWN/HIBERN8
2. **功耗等级** - 100% → 30% → 10% → 5% → 1%
3. **唤醒时间** - IDLE<10μs, SLEEP<50μs, HIBERN8<10ms
4. **HIBERN8 最重要** - 车载场景常用，需重点测试
5. **ICC 自适应** - 根据负载动态调整电流

### 测试设计要点
1. **基础测试** - 验证各模式功能正常
2. **循环测试** - 验证长期可靠性（1000 次循环）
3. **场景测试** - 模拟车载实际使用场景
4. **时序验证** - 严格验证唤醒时间要求
