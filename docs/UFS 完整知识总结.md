## 📑 第一部分：UFS 协议基础

### 1. UFS 3.1 架构

```
┌─────────────────────────────────────────────────────────┐
│                    UFS 系统架构                          │
├─────────────────────────────────────────────────────────┤
│  UFS Host ←→ UFS Interconnect ←→ UFS Device             │
│     ↓              ↓                    ↓               │
│  HCI 层        M-PHY 物理层          设备控制器          │
│     ↓              ↓                    ↓               │
│  应用层       UniPro 协议层          存储介质 (NAND)     │
└─────────────────────────────────────────────────────────┘
```

**关键参数**：
- 速率：11.6 Gbps/lane (HS-G4)
- 车道数：2 lane (读 + 写分离)
- 理论带宽：~2.9 GB/s
- LUN 数：最多 8 个

### 2. UFS 命令集

| 类型 | 命令 | 功能 |
|------|------|------|
| **SCSI 命令** | READ_10/16 | 数据读取 |
| | WRITE_10/16 | 数据写入 |
| | SYNCHRONIZE_CACHE | 刷缓存 |
| **UFS 命令** | QUERY_REQUEST | 查询描述符/属性 |
| | NOP_OUT | 链路检测 |
| **任务管理** | ABORT_TASK | 中止任务 |
| | CLEAR_ACA | 清除自动队列 |

### 3. 电源管理（5 种模式）

| Mode | 名称 | 功耗 | 唤醒时间 |
|------|------|------|---------|
| 0 | Active | 高 | 0 |
| 1 | Idle | 中 | <10μs |
| 2 | Sleep | 低 | <100μs |
| 3 | Power-Down | 极低 | <1ms |
| 4 | Deep-Stall | 最低 | <10ms |

### 4. 关键特性

**Write Booster**：
- 原理：使用 SLC NAND 作为缓存
- 效果：写入速度提升 2-3x
- 配置：通过 Query 命令开启/关闭

**HPB (Host Performance Booster)**：
- 原理：主机缓存 L2P 映射表
- 效果：随机读性能提升
- 要求：主机支持 HPB 功能

## 📑 第二部分：固件算法

### 1. FTL 基础

**核心功能**：
```
┌─────────────────────────────────────┐
│         FTL 三大核心算法            │
├─────────────────────────────────────┤
│  1. 地址映射（Address Mapping）     │
│  2. 垃圾回收（Garbage Collection）  │
│  3. 磨损均衡（Wear Leveling）       │
└─────────────────────────────────────┘
```

**地址映射类型**：
| 类型 | 粒度 | 优点 | 缺点 |
|------|------|------|------|
| Page-level | 页 | 随机写性能好 | 映射表大 |
| Block-level | 块 | 映射表小 | 随机写差 |
| Hybrid | 混合 | 平衡 | 复杂 |

### 2. 性能优化

**SLC Cache**：
- 原理：将 TLC 模拟为 SLC 使用
- 容量：通常为总容量的 5-10%
- 策略：写满后转为 TLC（速度下降）

**垃圾回收**：
- 触发条件：空闲块低于阈值
- 流程：选择victim 块 → 迁移有效数据 → 擦除
- 影响：写放大、性能波动

### 3. 可靠性

**ECC 纠错**：
- BCH/LDPC 编码
- 纠错能力：≥72bit/1KB
- 失败处理：RAID 恢复

**坏块管理**：
- 出厂坏块：标记并跳过
- 生长坏块：动态映射
- 阈值：RBER < 1000/1KB

## 📑 第三部分：3D NAND 特性

### 1. NAND 结构

```
Cell → Page → Block → Plane → Die → LUN
```

**典型参数**：
- Page 大小：16KB
- Block 大小：128 Pages
- Plane 数：2-4 per Die
- Die 数：4-8 per LUN

### 2. 失效模式

| 失效类型 | 原因 | 检测方法 |
|----------|------|---------|
| RTN (Random Telegraph Noise) | 电荷陷阱 | RBER 监控 |
| DCI (Disturbance) | 读/写干扰 | 错误计数 |
| Wear-out | 氧化层磨损 | P/E 循环计数 |
| Retention | 电荷泄漏 | 数据保持测试 |

### 3. 测试方法

**RBER (Raw Bit Error Rate)**：
- 公式：错误比特数 / 总比特数
- 阈值：<10^-5 (新盘)
- 告警：<10^-3

**UBER (Uncorrectable Bit Error Rate)**：
- ECC 后错误率
- 要求：<10^-15 (车规级)

### 1. 驱动架构

```
Block Device → UFS Driver → HCI → M-PHY
```

**关键文件**：
- `drivers/ufs/core/ufshcd.c` - 核心驱动
- `drivers/ufs/host/ufshcd-pltfrm.c` - 平台驱动
- `include/ufs/ufs.h` - 头文件

### 2. 设备树配置

```dts
&ufs {
    status = "okay";
    vcc-supply = <&vcc_ufs>;
    vccq-supply = <&vccq_ufs>;
    vccq2-supply = <&vccq2_ufs>;
};
```
