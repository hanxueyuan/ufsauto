## 2. 核心知识点回顾

### 2.1 UFS 系统架构（第 4 章）

**三层架构模型**：
```
Application Layer（应用层）
  ↕ SCSI/UFS 命令
Transport Layer（传输层 - UniPro）
  ↕ UPIU 数据包
Physical Layer（物理层 - M-PHY）
  ↕ 差分信号
```

**关键认知**：
- UFS 不是简单的存储接口，而是一个完整的协议栈
- Host 和 Device 之间通过 UPIU（UFS Protocol Information Unit）通信
- 支持最多 8 个 LUN，每个 LUN 独立寻址和管理
- Well-Known LUN（BOOT/RPMB/W-LU）提供特殊功能

### 2.2 命令协议（第 7 章）

**四大命令类别**：
1. **SCSI 命令**：READ_10/WRITE_10/SYNCHRONIZE_CACHE/UNMAP/INQUIRY/TEST UNIT READY 等
2. **UFS 原生命令**：QUERY REQUEST（读写描述符/属性/标志位）、NOP OUT/IN
3. **任务管理命令**：ABORT TASK/LUN RESET/QUERY TASK 等
4. **厂商命令**：Vendor-specific

**队列管理**：
- 最大队列深度 32（Tag 0-31）
- 支持乱序完成（Out-of-Order Completion）
- 对性能测试（QD 调优）至关重要

### 2.3 描述符体系（第 8 章）

**层次结构**：
```
Device Descriptor（设备级）
├── Configuration Descriptor（配置）
│   └── Unit Descriptor（LUN 级）
├── Geometry Descriptor（几何参数）
├── Health Descriptor（健康状态）
├── Interconnect Descriptor（互连信息）
└── String Descriptor（字符串）
```

**测试关键描述符**：
- **Health Descriptor**：bPreEOLInfo（寿命预警）+ bDeviceLifeTimeEstA/B（寿命估算）
- **Device Descriptor**：wSpecVersion（协议版本）+ bNumberLU（LUN 数量）
- **Unit Descriptor**：qLogicalBlockCount（容量）+ bMemoryType（存储类型）

### 2.4 电源管理（第 9 章）

**5 种电源模式**：

| 模式 | 相对功耗 | 唤醒时间 | 车载场景 |
|------|----------|----------|----------|
| ACTIVE | 100% | - | 行驶中 |
| IDLE | ~30% | <10μs | 短暂等待 |
| SLEEP | ~10% | <50μs | 停车等待 |
| POWERDOWN | ~5% | <1ms | 长时间停放 |
| HIBERN8 | ~1% | <10ms | 熄火休眠 |

**ICC 自适应电流控制**：
- 通过 bActiveICCLevel 属性控制
- 高级别 = 更大电流 = 更高性能 = 更高功耗
- 车规场景需要在性能和功耗间平衡

### 2.5 固件算法（通宵学习补充）

**FTL 三大核心算法**：
1. **地址映射**：逻辑地址（LBA）→ 物理地址（PBA），页映射/块映射/混合映射
2. **垃圾回收（GC）**：前台 GC（紧急，影响性能）和后台 GC（空闲时执行）
3. **磨损均衡（WL）**：动态 WL（热数据）和静态 WL（冷数据迁移）

**对测试的影响**：
- GC 导致写入延迟尖峰（P99/P99.99 延迟）
- WL 影响长期写入性能一致性
- SLC Cache 耗尽后性能骤降（需要测试 SLC Cache 边界）

## 7. 关键收获

### 技术认知

1. **UFS 不只是"快闪存"**：它是一个包含协议栈、电源管理、安全机制的完整存储子系统
2. **车规级要求远超消费级**：温度范围 -40~105°C、寿命 15 年、可靠性 1 DPPM
3. **电源管理是车载核心**：车辆熄火后存储设备的功耗直接影响电池寿命
4. **FTL 算法决定性能表现**：GC 和 WL 的实现质量决定了 P99 延迟和写入一致性
5. **测试要从协议层设计**：理解底层协议才能设计出有深度的测试用例

### 工程认知

1. **SysTest 框架已具备基本能力**：14 个测试脚本已重构完成，按 7 部分框架统一
2. **Precondition 检查已实现**：从实际系统抓取数据，不再是静态模板
3. **开发模式支持**：没有 UFS 设备也能开发和调试测试脚本
4. **文档驱动学习**：边学边写文档的效率远高于只看不记
