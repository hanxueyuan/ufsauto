## 1️⃣ M-PHY 物理层（第 10 章）

### 1.1 M-PHY 版本与速率

UFS 3.1 使用 **M-PHY v4.0** 规范

| Gear | 速率 (Gbps/lane) | 带宽 (2 lanes) | UFS 版本支持 |
|------|------------------|----------------|--------------|
| Gear 1 | 1.25 | 2.5 Gbps | UFS 2.0 |
| Gear 2 | 2.5 | 5.0 Gbps | UFS 2.0/2.1 |
| Gear 3 | 5.8 | 11.6 Gbps | UFS 3.0/3.1 |
| **Gear 4** | **11.6** | **23.2 Gbps** | **UFS 3.1/4.0** |

### 1.2 车道（Lane）配置

**UFS 支持 1 或 2 车道**：

```
单车道（1 Lane）:
  TX: 发送
  RX: 接收
  带宽：Gear 4 = 11.6 Gbps

双车道（2 Lanes）:
  TX0 TX1: 发送
  RX0 RX1: 接收
  带宽：Gear 4 = 23.2 Gbps
```

**车道绑定**：
- 双车道模式下，两条 lane 绑定使用
- 数据在两条 lane 上 interleaved 传输
- 提高带宽利用率

### 1.3 M-PHY 功耗状态

| 状态 | 名称 | 功耗 | 唤醒时间 | 说明 |
|------|------|------|----------|------|
| **ACTIVE** | 活跃状态 | 高 | - | 正常数据传输 |
| **STALL** | 暂停状态 | 中 | <1μs | 短暂空闲 |
| **HIBERN8** | 休眠状态 | 极低 | <10ms | 长时间空闲 |

**状态转换**：
```
ACTIVE ←→ STALL ←→ HIBERN8
```

### 1.4 M-PHY 编码方式

**8b/10b 编码**（Gear 1-3）：
- 8 位数据 → 10 位编码
- 开销：20%
- 优点：DC 平衡，时钟恢复容易

**128b/150b 编码**（Gear 4）：
- 128 位数据 → 150 位编码
- 开销：17.2%
- 优点：更高效率，适合高速

**有效数据速率计算**：
```
Gear 4 原始速率：11.6 Gbps/lane
128b/150b 编码效率：128/150 = 85.3%
有效速率：11.6 × 0.853 = 9.9 Gbps/lane
双车道带宽：9.9 × 2 = 19.8 Gbps ≈ 2.475 GB/s
```

## 2️⃣ UniPro 协议层（第 10 章）

### 2.1 UniPro 层次结构

```
┌─────────────────────────────────────────────────────────┐
│                    UniPro 协议栈                          │
├─────────────────────────────────────────────────────────┤
│  L4 应用层        → UFS 协议层                            │
│  L3.5 传输层      → 端到端流量控制                        │
│  L3 网络层        → 路由、寻址                            │
│  L2 数据链路层    → 链路管理、错误检测                     │
│  L1.5 物理适配层  → M-PHY 控制                            │
│  L1 物理层        → M-PHY                                 │
└─────────────────────────────────────────────────────────┘
```

### 2.2 UniPro 数据包结构

```
┌─────────────────────────────────────────────────────────┐
│                    UniPro 数据包                         │
├──────────┬──────────┬──────────┬──────────┬─────────────┤
│  Header  │  Sequence│  Data    │  CRC     │  Footer     │
│  (4B)    │  (2B)    │  (可变)   │  (2B)    │  (2B)       │
└──────────┴──────────┴──────────┴──────────┴─────────────┘
```

**字段说明**：
| 字段 | 大小 | 功能 |
|------|------|------|
| Header | 4 字节 | 包类型、源/目的地址 |
| Sequence | 2 字节 | 序列号（用于流量控制） |
| Data | 可变 | 有效载荷（命令/数据） |
| CRC | 2 字节 | 循环冗余校验 |
| Footer | 2 字节 | 包结束标志 |

### 2.3 UniPro 流量控制

**基于信用的流量控制**：
```
1. 接收方初始化信用值（如 100）
2. 发送方每发送 1 个包，信用值 -1
3. 信用值=0 时，停止发送
4. 接收方处理完成后，发送信用更新
5. 发送方恢复发送
```

**优点**：
- 防止接收方缓冲区溢出
- 保证数据不丢失
- 支持 QoS（优先级流量）

## 3️⃣ 时钟与信号（第 11 章）

### 3.1 参考时钟频率

UFS 支持 4 种参考时钟频率：

| 频率 | 容差 | 典型应用 |
|------|------|----------|
| **19.2 MHz** | ±10 ppm | 移动设备 |
| **26 MHz** | ±10 ppm | 移动设备 |
| **38.4 MHz** | ±10 ppm | 高性能设备 |
| **52 MHz** | ±10 ppm | 高性能设备 |

**时钟源**：
- 主机提供时钟（RefClk）
- 设备使用主机时钟或内部 PLL

### 3.2 时钟频率切换

**动态时钟频率切换**：
```
1. 主机请求切换频率
2. 设备确认
3. 等待时钟稳定
4. 恢复正常操作
```

**应用场景**：
- 高性能模式 → 52 MHz
- 省电模式 → 19.2 MHz

### 3.3 时钟精度要求

| 参数 | 要求 | 说明 |
|------|------|------|
| 频率容差 | ±10 ppm | 百万分之十 |
| 抖动（Jitter） | <10 ps RMS | 均方根值 |
| 相位噪声 | <-100 dBc/Hz @ 10kHz | 偏移 10kHz 处 |

| 纹波 | <100 mV | 峰峰值 |

#### I/O 电源（VCCQ）

| 参数 | 值 | 说明 |
|------|-----|------|
| 电压范围 | 1.1 - 1.3 V **或** 1.7 - 1.95 V | 双电压支持 |
| 标称电压 | 1.2 V **或** 1.8 V | 二选一 |
| 纹波 | <50 mV | 峰峰值 |

#### 辅助 I/O 电源（VCCQ2）

| 参数 | 值 | 说明 |
|------|-----|------|
| 电压范围 | 1.1 - 1.3 V **或** 1.7 - 1.95 V | 与 VCCQ 独立 |

### 4.2 功耗等级

| 模式 | VCC 电流 | VCCQ 电流 | 总功耗 |
|------|----------|-----------|--------|
| **ACTIVE**（读） | ~200 mA | ~50 mA | ~700 mW |
| **ACTIVE**（写） | ~250 mA | ~50 mA | ~850 mW |

| **SLEEP** | ~5 mA | ~5 mA | ~30 mW |
| **HIBERN8** | ~1 mA | ~1 mA | ~6 mW |

**注**：实际功耗取决于具体实现和负载

## 5️⃣ 测试与测量（第 13 章）

### 5.1 一致性测试

#### 协议一致性测试

| 测试项 | 测试方法 | 验收标准 |
|--------|----------|----------|
| 命令格式 | 发送各种命令 | 符合 UPIU 格式 |

| 错误处理 | 发送无效命令 | 正确返回错误 |

#### 电气一致性测试

| 测试项 | 测试方法 | 验收标准 |
|--------|----------|----------|
| 电源质量 | 测量电压纹波 | <100 mV (VCC), <50 mV (VCCQ) |
| 信号质量 | 眼图测试 | 眼图张开度>70% |
| 阻抗测试 | TDR 测量 | 100Ω±10% (差分) |
| 时序测试 | 示波器测量 | 符合时序要求 |

### 5.2 性能测试

#### 带宽测试

```
测试配置：
- 主机：UFS Host Controller
- 设备：UFS Device
- 线缆：阻抗匹配测试线

测试方法：
1. 顺序读取 1GB 数据
2. 记录时间和数据量
3. 计算带宽 = 数据量 / 时间

验收标准：
- Gear 4, 2 lanes: ≥2.0 GB/s
- Gear 3, 2 lanes: ≥1.0 GB/s
```

#### 延迟测试

```
测试配置：
- 4K 随机读取
- 队列深度 QD1

测试方法：
1. 发送 10000 次 4K 读取命令
2. 记录每次命令响应时间
3. 统计 P50/P99/P99.99 延迟

验收标准：
- P50: <100 μs
- P99: <200 μs
- P99.99: <500 μs
```

### 5.3 功耗测试

```
测试配置：
- 高精度电流表（uA 级）
- 可编程电源
- 示波器（捕捉瞬态）

测试方法：
1. 测量各电源模式电流
2. 计算功耗 = V × I
3. 测量模式转换时间

验收标准：
- ACTIVE 读：≤800 mW
- ACTIVE 写：≤900 mW
- HIBERN8: ≤10 mW
```

## 📝 学习总结

### 核心要点

1. **M-PHY v4.0** - UFS 3.1 使用 M-PHY v4.0，Gear 4 速率 11.6 Gbps/lane
2. **双车道绑定** - 支持 2 lanes，总带宽 23.2 Gbps
3. **128b/150b 编码** - Gear 4 使用高效编码，效率 85.3%
4. **UniPro 协议栈** - L1-L4 分层架构，流量控制基于信用
5. **4 种参考时钟** - 19.2/26/38.4/52 MHz，容差±10 ppm
6. **双电压 I/O** - VCCQ 支持 1.2V 或 1.8V
7. **功耗等级** - ACTIVE ~700mW，HIBERN8 ~6mW

### 测试应用

1. **物理层测试** - 眼图、阻抗、时序测试
2. **协议一致性测试** - 命令格式、响应格式、错误处理
3. **性能测试** - 带宽、延迟、IOPS
4. **功耗测试** - 各模式功耗测量、模式转换时间
