# FTL 算法测试用例设计

**创建时间**: 2026-03-13  
**基于**: OpenSSD 开源项目学习  
**状态**: 持续更新中

---

## 📋 测试分类总览

```
┌─────────────────────────────────────────────────────────┐
│              FTL 测试体系                                │
├─────────────────────────────────────────────────────────┤
│  1. 地址映射测试    → 映射表读写/持久化/恢复            │
│  2. 磨损均衡测试    → 动态 WL/静态 WL/擦除分布          │
│  3. 垃圾回收测试    → GC 触发/Victim 选择/性能影响      │
│  4. 坏块管理测试    → 工厂坏块/生长坏块/数据迁移        │
│  5. 掉电恢复测试    → 映射表恢复/数据完整性             │
│  6. 性能测试        → 读写性能/延迟/GC 影响              │
└─────────────────────────────────────────────────────────┘
```

## 1️⃣ 地址映射测试

| 用例 ID | 测试项 | 测试方法 | 验收标准 | 优先级 |

| MAP-003 | 单页映射写入 | 写入 LBA 0 → 查映射表 → 验证 PBA 更新 | PBA 已更新 | 🔴 高 |
| MAP-004 | 多页映射写入 | 写入 LBA 0-255 → 验证映射批量更新 | 全部更新 | 🔴 高 |
| MAP-005 | 映射表边界测试 | 读取 LBA Max/Max-1 | 边界正确处理 | 🟠 中 |
| MAP-006 | 无效 LBA 读取 | 读取未映射 LBA | 返回错误 | 🟠 中 |

| MAP-102 | 映射表恢复 | 掉电后重启 → 从 NAND 恢复映射表 → 验证数据可访问 | 恢复成功 | 🔴 高 |
| MAP-103 | 日志式更新 | 记录映射表变更日志 → 验证日志完整性 | 日志完整 | 🟠 中 |
| MAP-104 | 映射表一致性 | 随机掉电 → 验证映射表一致性 | 无损坏 | 🔴 高 |

## 3️⃣ 垃圾回收测试

### 3.1 GC 触发测试

| 用例 ID | 测试项 | 测试方法 | 验收标准 | 优先级 |

| GC-002 | 前台 GC 触发 | 写入时无可用块 → 验证前台 GC | GC 触发 | 🔴 高 |
| GC-003 | 后台 GC 触发 | 空闲时 → 验证后台 GC | GC 触发 | 🟠 中 |
| GC-004 | GC 阈值配置 | 配置不同 GC 阈值 → 验证触发点 | 符合配置 | 🟢 低 |

### 3.2 Victim Block 选择测试

| 用例 ID | 测试项 | 测试方法 | 验收标准 | 优先级 |
|---------|--------|----------|----------|--------|
| GC-101 | 无效页统计 | 统计各块无效页数量 | 统计准确 | 🔴 高 |
| GC-102 | Victim 选择 | 选择无效页最多的块 | 选择正确 | 🔴 高 |
| GC-103 | 多块对比 | 对比不同块的无效页比例 | 选择最优 | 🟠 中 |
| GC-104 | 边界情况 | 所有块无效页相同 → 验证选择策略 | 合理选择 | 🟢 低 |

### 3.3 GC 执行测试

| 用例 ID | 测试项 | 测试方法 | 验收标准 | 优先级 |
|---------|--------|----------|----------|--------|
| GC-201 | 有效页迁移 | 读取 Victim 块有效页 → 写入新块 → 验证数据 | 数据一致 | 🔴 高 |
| GC-202 | 映射表更新 | GC 后验证映射表指向新位置 | 映射正确 | 🔴 高 |
| GC-203 | Victim 擦除 | GC 后验证 Victim 块已擦除 | 已擦除 | 🔴 高 |
