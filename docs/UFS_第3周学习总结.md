## 2. 关键收获

### 2.1 FTL 是性能和寿命的核心

本周最大的认知升级：**FTL 算法的质量直接决定了 UFS 设备的性能表现和使用寿命**。

- **映射方式**决定了随机写性能上限：页映射最优但 RAM 占用大，DFTL 是最佳折中
- **GC 算法**决定了 P99 延迟：前台 GC 是写入延迟尖峰的根本原因
- **WL 算法**决定了设备寿命：没有良好的 WL，设备可能在 50% P/E 时就失效
- **坏块管理**决定了可靠性底线：Reserved Pool 不足会导致设备提前报废

### 2.2 WAF 是连接一切的关键指标

```
WAF 影响链:
  WAF ↑ → NAND 写入量 ↑ → 擦除次数 ↑ → 寿命 ↓
  WAF ↑ → GC 频率 ↑ → 前台 GC ↑ → P99 延迟 ↑
  WAF ↑ → 功耗 ↑ → 发热 ↑ → NAND 退化加速

降低 WAF 的手段:
  1. 增加 OP 比例（最有效）
  2. 优化 GC Victim 选择（CAT 算法）
  3. Host 端 UNMAP/Trim 通知
  4. 顺序写入优先（WAF≈1.0）
```

### 2.3 车规级的特殊挑战

128GB TLC 在 ADAS 场景下的寿命计算揭示了一个严峻现实：

- 每日写入 1.46 TB × WAF 2.0 = 2.92 TB 实际 NAND 写入
- 15 年需要 111,000 次 P/E，远超 TLC 的 3000 次限制
- **必须**通过增大容量、优化 WAF、限制写入量来解决
