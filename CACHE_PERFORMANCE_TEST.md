# Cache性能和磨损均衡测试方案

## 🎯 测试目标
验证UFS的缓存策略、磨损均衡算法、数据搬移机制，全面掌握底层行为特性。

---
## 📋 测试内容
### 1. 缓存性能测试
#### 1.1 Data Cache大小测试
**测试方法**：
1. 写入不同大小的测试文件：128MB/256MB/512MB/1GB/2GB/4GB
2. 写入完成后立即读取，记录写入速度和读取速度
3. 读取速度远高于写入速度的最大文件大小即为缓存大小
4. 重复测试3次，取平均值

#### 1.2 缓存命中率测试
**测试方法**：
1. 随机读取4KB小文件，总数据量分别为缓存大小的50%/100%/200%
2. 统计不同数据量下的读取性能和命中率
3. 对比缓存开启/关闭（如果支持配置）的性能差异

#### 1.3 缓存刷新策略测试
**测试方法**：
1. 写入满缓存大小的数据，然后分别间隔不同时间读取：
   - 立即读取（0秒）
   - 间隔10秒读取
   - 间隔30秒读取
   - 间隔1分钟读取
   - 间隔5分钟读取
   - 间隔10分钟读取
   - 间隔30分钟读取
   - 间隔1小时读取
2. 记录每个时间点的读取速度，观察缓存刷新时间点
3. 分析缓存数据的保留时间和刷新策略

---
### 2. 磨损均衡（Wear Leveling）算法测试
#### 2.1 静态磨损均衡测试
**测试方法**：
1. 写入大文件占满大部分空间，保留少量空闲空间
2. 持续写入小文件到空闲区域，反复擦写同一逻辑块
3. 定期读取之前写入的静态大文件，检查数据完整性
4. 监控坏块数量变化，验证静态数据是否被自动搬移

#### 2.2 动态磨损均衡测试
**测试方法**：
1. 持续随机写入整个磁盘空间，持续运行72小时
2. 定期读取UFS健康信息，记录擦除次数分布
3. 验证擦除次数的均匀性，最大最小擦除次数差≤20%

---
### 3. 垃圾回收和数据搬移测试
#### 3.1 垃圾回收触发阈值测试
**测试方法**：
1. 占满磁盘90%空间，删除部分文件产生碎片化空闲空间
2. 持续写入新数据，监控写入性能变化
3. 观察性能突然下降的时间点，即为垃圾回收触发阈值
4. 记录垃圾回收持续时间和性能恢复情况

#### 3.2 空闲时垃圾回收测试
**测试方法**：
1. 占满磁盘90%空间，删除30%文件产生碎片化
2. 系统进入空闲状态，监控磁盘IO活动
3. 记录垃圾回收开始时间、持续时间、CPU占用
4. 垃圾回收完成后测试读写性能恢复情况

### 4. 稳态性能测试
#### 4.1 长时间稳态性能
**测试方法**：
1. 连续24小时满负载随机写入，记录每秒IOPS和延迟
2. 绘制性能时间曲线，分析稳态性能波动
3. 统计平均性能、p99延迟、最大/最小性能
**验收标准**：
- 稳态平均性能≥标称值的90%
- 性能波动≤15%
- 无IO错误、无掉盘

## 🔒 安全保障
1. 所有测试都使用固定大小的测试文件，总占用空间≤20GB
2. 测试前自动检查可用空间≥30GB，测试后自动清理所有文件
3. 实时监控剩余空间，低于10GB自动终止测试
4. 长时间测试支持断点续测，避免重复劳动
