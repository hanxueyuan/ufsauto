**错误 2**: rand_write_burst 异常
```
2026-04-09 08:15:10 - ERROR - ✗ rand_write_burst 异常：Expecting value: line 1 column 1 (char 0)
```
**原因**: FIO JSON 输出解析失败  
**影响**: 随机写测试无法获取结果

**错误 3**: mixed_rw 异常
```
2026-04-09 08:15:16 - ERROR - ✗ mixed_rw 异常：Expecting value: line 1 column 1 (char 0)
```
**原因**: FIO JSON 输出解析失败  
**影响**: 混合读写测试无法获取结果

### 根本原因分析

**问题**: FIO 的 JSON 输出可能被其他输出（如警告信息）污染

```
FIO 输出:
note: both iodepth >= 1 and synchronous I/O engine are selected, queue depth will be capped at 1
{
  "fio version" : "fio-3.36",
  ...
}
```

FIO 在 JSON 输出前打印了一条警告，导致 JSON 解析失败。

### 解决方案

修改 `verify_all_tests.py`，合并 stdout 和 stderr 并智能提取 JSON：

```python
# 合并输出（FIO 可能输出到任一位置）
combined_output = result.stdout + result.stderr

# 从合并输出中查找 JSON
json_start = combined_output.find('{')
json_end = combined_output.rfind('}') + 1
if json_start >= 0 and json_end > json_start:
    json_str = combined_output[json_start:json_end]
    fio_output = json.loads(json_str)
```
