## 🎯 核心功能

### 1. 统一入口

```bash
systest <command> [options]

命令:
  run       - 执行测试
  list      - 列出可用测试
  report    - 生成/查看报告
  analyze   - 失效分析
  config    - 配置管理
```

### 2. 参数化执行

```bash
# 执行套件
systest run -s performance -d /dev/ufs0

# 执行单个测试
systest run -t seq_read_burst -v

# 干跑模式
systest run -s performance --dry-run
```

### 3. 自动报告

- **HTML 报告**: 可视化测试结果、系统信息、设备信息
- **JSON 结果**: 结构化数据，便于后续处理
- **文本摘要**: 快速查看测试概况

### 4. 失效分析

三层分析架构：
1. **规则匹配** - 基于预定义失效模式库
2. **关联分析** - 多因素综合分析
3. **根因定位** - 置信度排序，输出建议

## 📝 开发笔记

### 设计原则

1. **纯标准库** - 确保开发板可直接运行，无需 pip 安装
2. **模块化** - 核心功能独立，易于扩展
3. **配置驱动** - 测试参数、目标值通过配置调整
4. **渐进式** - 从干跑到实际执行，从基础到高级

### 关键实现

1. **FIO 集成** - 通过 subprocess 调用，JSON 格式解析
2. **规则引擎** - 基于字典的失效规则定义，支持置信度评估
3. **报告模板** - 内联 HTML，无需外部模板文件
4. **干跑模式** - 显示将要执行的 FIO 命令和验收目标，便于审查

### 验证发现 (2026-03-16)

1. **失效分析有效** - 成功识别 SLC Cache 耗尽、GC 干扰等失效模式
2. **置信度评估合理** - 基于证据数量和规则置信度综合计算
3. **报告格式完整** - HTML 可视化、JSON 结构化、TXT 摘要三种格式
4. **干跑模式实用** - 可在执行前审查测试配置和 FIO 命令

### 已知限制

1. 需要 FIO 工具支持
2. 需要 root 权限访问 UFS 设备
3. 失效分析规则需要实际测试数据验证和优化
4. 模拟测试无法验证 FIO 实际执行和硬件交互
