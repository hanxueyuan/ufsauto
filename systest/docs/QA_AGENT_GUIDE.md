# QA Agent 配置指南

**配置日期**: 2026-03-16  
**职责**: 自动查看 CI/CD 结果并生成报告  
**汇报渠道**: 飞书

---

## 📋 QA Agent 职责

### 主要职责

1. **自动监控 CI/CD 结果**
   - 监听 SysTest CI/CD workflow 完成事件
   - 自动下载 CI/CD 报告
   - 分析检查结果

2. **生成 QA 摘要报告**
   - 提取关键信息（状态/失败项/链接）
   - 生成简洁的摘要报告
   - 提供下一步行动建议

3. **自动汇报**
   - 飞书消息通知
   - 包含关键信息和链接
   - 区分成功/失败状态

---

## 🛠️ 配置说明

### Workflow 配置

**文件**: `.github/workflows/qa-agent.yml`

**触发条件**:
```yaml
on:
  workflow_run:
    workflows: ["SysTest CI/CD"]
    types:
      - completed
```

**执行流程**:
```
CI/CD 完成
    ↓
[下载 CI/CD 报告]
    ↓
[读取检查结果]
    ↓
[生成 QA 摘要]
    ↓
[上传 QA 报告]
    ↓
[发送汇报通知]
```

## 🔍 QA 审查要点

### 成功情况

**检查项全部通过**:
- ✅ 代码可以合并
- ✅ 可以继续部署
- ✅ 无需人工干预

### 失败情况

**部分检查失败**:
- ❌ 需要查看失败原因
- ❌ 需要修复后重新运行
- ❌ 可能需要人工干预

### 常见失败原因

1. **代码质量检查失败**
   - Flake8 发现语法错误
   - Black 格式检查不通过
   - Isort 导入顺序错误

2. **测试用例验证失败**
   - 命名规范不符合（应该用驼峰命名）
   - 注释字段缺失
   - 配置文件格式错误

3. **文档验证失败**
   - 必要文档缺失
   - 文档格式错误

## ⚠️ 注意事项

### CI/CD 环境限制

**GitHub Actions 环境没有 UFS 硬件设备**：
- ✅ 执行代码质量检查、配置验证、最小化验证
- ❌ 不执行实际的 UFS 性能测试
- ✅ 实际硬件测试在开发板上执行

### QA Agent 职责范围

**QA Agent 负责**：
- ✅ 查看 CI/CD 结果
- ✅ 生成 QA 摘要
- ✅ 发送汇报通知

**QA Agent 不负责**：
- ❌ 实际执行测试
- ❌ 修复代码问题
- ❌ 决定代码是否合并
