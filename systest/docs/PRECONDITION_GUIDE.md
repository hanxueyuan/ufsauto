## 🎯 Precondition 的正确理解

### Precondition 是什么？

**Precondition 不是写死的具体值，而是说明脚本会：**
1. **收集哪些信息** - 通用的，所有测试都要收集
2. **检查哪些配置** - 根据测试项目选择性检查

---

## 📋 Precondition 结构

### 1. 收集的信息（通用，所有测试都收集）

```python
Precondition:
1.1 系统环境收集
    - 操作系统版本
    - CPU/内存配置
    - FIO 工具版本

1.2 测试目标信息收集
    - 设备路径
    - 设备型号
    - 固件版本
    - 设备容量
    - 可用空间

1.3 存储设备配置检查
    - 需要开启的功能
    - 需要关闭的功能
    - 特殊配置项

1.4 UFS 器件配置检查
    - LUN 数量
    - 各 LUN 配置
    - LUN 映射关系

1.5 器件健康状况检查
    - SMART 状态
    - 剩余寿命
    - 坏块数量
    - 温度状态
    - 错误计数

1.6 前置条件验证
    - 必须满足的条件列表
```

### 2. 配置检查（根据测试项目选择性检查）

#### Performance - Burst 测试
```python
1.3 存储设备配置检查
    - 开启功能：TURBO Mode（提升峰值性能）
    - 关闭功能：省电模式（避免性能限制）
    - 特殊配置：无
```

#### Performance - Sustained 测试
```python
1.3 存储设备配置检查
    - 开启功能：无特殊开启
    - 关闭功能：省电模式（避免性能限制）
    - 特殊配置：无

1.6 前置条件验证
    - ✓ 可用空间必须≥20GB（长时间测试需要更多空间）
```

#### Reliability - 长期稳定性测试
```python
1.3 存储设备配置检查
    - 开启功能：无特殊开启
    - 关闭功能：自动休眠（避免影响长时间测试）
    - 特殊配置：IO 调度器设置为 none（减少调度延迟）

1.6 前置条件验证
    - ✓ 电源必须稳定（建议使用 UPS）
    - ✓ 散热条件良好（建议加散热片或风扇）
```

#### Scenario - 场景测试
```python
1.3 存储设备配置检查
    - 开启功能：无特殊开启
    - 关闭功能：省电模式（避免性能限制）
    - 特殊配置：无

1.2 测试目标信息收集
    - 可用空间：≥5GB（根据测试数据量决定）
```

## ✅ 正确的注释示例

### 示例 1：Performance Burst 测试

```python
Precondition:
1.1 系统环境收集
    - 操作系统：Debian 12, kernel 5.15.120
    - CPU/内存：8 核，16GB
    - FIO 版本：fio-3.33

1.2 测试目标信息收集
    - 设备路径：/dev/ufs0
    - 设备型号：UFS 3.1 128GB
    - 固件版本：v1.0.0
    - 设备容量：128GB
    - 可用空间：≥10GB

1.3 存储设备配置检查
    - 开启功能：TURBO Mode（提升峰值性能）
    - 关闭功能：省电模式（避免性能限制）
    - 特殊配置：无

1.4 UFS 器件配置检查
    - LUN 数量：4 个
    - LUN0：64GB 系统盘（已挂载）
    - LUN1：32GB 数据盘（测试目标）
    - LUN2：16GB 日志盘
    - LUN3：16GB 预留
    - LUN 映射：LUN1→/dev/ufs0

1.5 器件健康状况检查
    - SMART 状态：正常
    - 剩余寿命：98%
    - 坏块数量：0
    - 温度状态：35℃（当前）/ 45℃（最高）
    - 错误计数：CRC 错误=0, 重传次数=0

1.6 前置条件验证
    - ✓ SMART 状态必须为正常
    - ✓ 可用空间必须≥10GB
    - ✓ 温度必须<70℃
    - ✓ 剩余寿命必须>90%
```

## 📝 配置检查说明

### 通用检查（所有测试）

| 检查项 | 说明 | 默认要求 |
|--------|------|---------|
| SMART 状态 | 器件健康状态 | 正常 |
| 可用空间 | 剩余存储空间 | ≥10GB |
| 温度 | 器件工作温度 | <70℃ |
| 剩余寿命 | NAND 剩余寿命 | >90% |

### 特殊检查（根据测试类型）

| 测试类型 | 特殊检查 | 说明 |
|---------|---------|------|
| **Burst** | TURBO Mode 开启 | 提升峰值性能 |
| **Sustained** | 可用空间≥20GB | 长时间测试需要更多空间 |
| **Reliability** | 关闭自动休眠 | 避免影响长时间测试 |
| **Reliability** | IO 调度器=none | 减少调度延迟 |
| **Reliability** | 电源稳定 + 散热良好 | 24 小时测试需要 |
| **Scenario** | 可用空间≥5GB | 根据测试数据量决定 |

## ⚠️ 常见错误

### ❌ 错误 1：Precondition 写死具体值

```python
# 错误示例
Precondition:
- 操作系统：Debian 12, kernel 5.15.120  # 不应该写死
- 设备型号：UFS 3.1 128GB  # 不应该写死
```

**正确做法**：说明会收集这些信息，具体值由脚本运行时收集。

### ❌ 错误 2：Precondition 重复

```python
# 错误示例
Precondition:
1.1 系统环境收集
    ...

Precondition:  # 不应该重复
1.1 系统环境收集
    ...
```

**正确做法**：Precondition 只出现一次。

### ❌ 错误 3：缺少配置检查说明

```python
# 错误示例
Precondition:
1.3 存储设备配置检查
    - 特殊配置：无  # 没有说明为什么
```

**正确做法**：说明配置检查的目的，如"关闭自动休眠（避免影响长时间测试）"。

---

## 🎯 总结

**Precondition 注释应该**：
1. ✅ 说明脚本会收集哪些信息（通用）
2. ✅ 说明脚本会检查哪些配置（根据测试项目）
3. ✅ 说明配置检查的目的
4. ✅ 使用统一的分级格式（1.1/1.2/1.3...）
5. ✅ 避免写死具体值（具体值由脚本运行时收集）

**Precondition 注释不应该**：
1. ❌ 写死具体的操作系统、设备型号等
2. ❌ 重复 Precondition 部分
3. ❌ 缺少配置检查说明
4. ❌ 使用不统一的格式
